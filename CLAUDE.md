# CLAUDE.md

Guidance for working in this repo.

## What this is

Marketing site for EMJU (a consulting firm). Two pages: front page (`/`) and contact
(`/contact`). No auth. TanStack Start (React 19) on Cloudflare Workers.

## Conventions

- **Content is data.** All copy lives in `src/content/site.ts`. Edit copy there, not in JSX.
- **Fonts:** use the `font-display` class (Instrument Serif) for headings, `font-ui`
  (Instrument Sans) for labels/nav/buttons. Body text defaults to Atkinson Hyperlegible.
- **Styling:** Tailwind v4 + daisyUI v5. The theme (`emju` / `emju-dark`) is defined in
  `src/styles.css`. Prefer daisyUI semantic colors (`primary`, `base-100`, `neutral`, …).
- **Server-only code** goes under `src/server/`. Read env via `getEnv()` (which wraps
  `cloudflare:workers`), never `process.env` at module scope. Keep DB / secrets inside
  server functions, never in loaders or components.
- **Validation** that must run on client + server lives in `src/lib` (pure, no imports of
  server modules) so it stays unit-testable.

## Testing

- Add `data-testid` (or `aria-label`) markers to new interactive elements so scenes can
  target them by intent.
- Unit tests: co-located `*.test.ts` in `src/lib`. E2E scenes: `scenetest/scenes/*.spec.md`.
- After a fresh `openTo`, allow for hydration before clicking handler-driven controls (the
  contact scene uses a short `wait`).

## Gate before committing

`pnpm format && pnpm lint && pnpm typecheck && pnpm test && pnpm build`
(and `pnpm scene` against a running `pnpm dev` when touching the UI/contact flow).

## Gotchas

- `worker-configuration.d.ts` is generated by `wrangler types` (runs on `postinstall`); it's
  gitignored. Run `pnpm cf-types` after changing `wrangler.jsonc`.
